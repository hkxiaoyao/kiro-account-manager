# 代码质量规范

## 核心原则

**精简代码 ≠ 简化代码**

- 精简 = 减少重复、合并相似逻辑、消除冗余
- 简化 = 降低功能复杂度（不是目标）

## 重复代码处理

### 识别重复
- 相同的 3 行以上代码出现 2 次以上 → 提取函数
- 相似的逻辑结构（只有参数不同）→ 参数化合并
- 相同的错误处理模式 → 提取通用处理函数

### 提取原则
- 提取的函数要有明确的单一职责
- 函数名要能描述其功能
- 不要为了提取而提取，2 行代码没必要单独成函数

## 代码审查检查项

1. 是否有重复代码块？
2. 是否有相似的 if-else / match 分支可以合并？
3. 是否有多处相同的错误处理？
4. 是否有未使用的变量/导入？
5. 是否有可以用迭代器链简化的循环？

## 修改后必做

- 运行 `cargo check`（Rust）或对应的类型检查
- 确认无编译错误再继续
- 清理警告（unused 变量等）

## 禁止事项

- ❌ 不要为了拆分而拆分文件
- ❌ 不要增加不必要的抽象层
- ❌ 不要引入新的重复代码
- ❌ 不要在没有验证编译的情况下继续修改
- ❌ 不要一味地检查重复代码，要结合实际情况判断是否需要提取
